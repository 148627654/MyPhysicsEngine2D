### V2 性能巅峰版：15天进阶进度表

#### 第1阶段：空间换时间——动态 AABB 树（第1-6天）
*   **第1-2天：树结构与叶子节点管理**
    *   实现 `DynamicAABBTree` 类。
    *   设计 `Node` 结构（包含 AABB、父节点索引、左右子节点）。
    *   实现**节点的动态插入**：寻找最佳同级节点（基于表面积启发式算法 SAH 的简化版）。
*   **第3天：树的自我平衡（Tree Rotations）**
    *   实现类似 AVL 树的**旋转逻辑**。
    *   当插入或移动导致树不平衡时，通过旋转维持树的高度，确保查询效率始终为 $O(\log N)$。
*   **第4天：肥包围盒（Fat AABB）与移动更新**
    *   实现 **Fat AABB**：为物体的 AABB 增加一个微小的位移缓冲区（Buffer）。
    *   **优化逻辑**：只有当物体移出这个“肥包围盒”时，才在树中更新节点位置，极大减少树重构频率。
*   **第5天：树查询（Query）与相交检测**
    *   实现 `Query(AABB)` 接口：返回所有可能相交的候选对。
    *   实现 `RayCast(Ray)` 接口：在树中快速进行射线检测。
*   **第6天：宽相集成（Broadphase Integration）**
    *   在 `World::Step` 中移除 $O(N^2)$ 的双重循环。
    *   集成 `DynamicAABBTree` 到宽相阶段。
    *   **验证**：对比 V1 和 V2 在 500 个物体下的帧率差异。

#### 第2阶段：能量治理——睡眠机制与岛屿（第7-10天）
*   **第7天：物体岛屿（Islands）构建**
    *   实现 **BFS/DFS 搜索**：将互相接触的物体通过“约束边”连接成一个独立的“岛屿”。
    *   **意义**：解算器不再对整个世界进行解算，而是分岛屿并行处理。
*   **第8-9天：休眠算法（Sleeping Logic）**
    *   为 `Body` 增加睡眠计时器。
    *   **判定条件**：如果岛屿内所有物体的线速度和角速度在 0.5 秒内持续低于阈值，则全岛进入休眠。
    *   休眠物体不参与碰撞检测和冲量解算，CPU 占用降为 0。
*   **第10天：唤醒系统（Wake-up Triggers）**
    *   实现唤醒逻辑：当一个运动物体撞击休眠物体，或休眠物体受力时，立即唤醒整个岛屿。

#### 第3阶段：高速防御——持续碰撞检测 CCD（第11-13天）
*   **第11天：扫掠包围盒（Swept AABB）**
    *   计算物体从 $t$ 到 $t+dt$ 时间段内扫过的轨迹包围盒。
    *   用于宽相初步筛选可能发生“穿墙（Tunneling）”的高速物体。
*   **第12天：撞击时间（TOI - Time of Impact）**
    *   针对高速运动对，计算它们在帧内发生碰撞的精确时间 $t \in [0, 1]$。
    *   **数学挑战**：实现简单的线性 TOI 或二分法寻找撞击点。
*   **第13天：子步解算（Sub-stepping）**
    *   当检测到穿墙预兆时，将该对物体回溯到 $TOI$ 时间点，进行微步积分和冲量处理，彻底杜绝“穿墙”现象。

#### 第4阶段：工程验证与巅峰对决（第14-15天）
*   **第14天：基准测试工具（Benchmark Tool）**
    *   编写 `Profiler` 类：精确记录 `Broadphase`、`Narrowphase`、`Solver` 各阶段耗时。
    *   创建“压力场景”：如 2000 个圆球坠落、50 层高的方块塔。
*   **第15天：V1 vs V2 性能报告**
    *   导出数据：生成横向对比 CSV。
    *   **可视化亮点**：在 README 中贴出“性能增长曲线图”，展示在物体数量增加时，V2 如何保持帧率平稳。
