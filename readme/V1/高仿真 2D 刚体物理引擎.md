---

### 如何验证你的物理引擎在工作？（没有GUI怎么看结果？）
1.  **文本日志 (Logging)**：在控制台实时打印：`Frame 10: Box A at (10.2, 5.4), Box B at (2.1, 0.0)`。
2.  **CSV 导出**：将每帧的数据写入 `.csv` 文件，完成后用 Excel 选定数据插入“散点图”，你会看到完美的抛物线或反弹轨迹。
3.  **单元测试 (Unit Testing)**：这是大杀器。编写测试用例验证：动量守恒吗？能量损耗对吗？

### 15天纯逻辑版进度表

#### 第1阶段：核心数学与基础结构（第1-3天）
*   **第1天：Vec2 数学库**
    *   实现 `Vector2` 类（加减乘除、点积、叉积、长度）。
    *   **C++11 练习**：使用 `constexpr` 优化数学常量，使用 `operator overloading`。
*   **第2天：物理状态与积分器**
    *   创建 `RigidBody` 类。属性：`position`, `velocity`, `force`, `mass`, `friction` 等。
    *   实现 `Step(float dt)` 函数，使用 **半隐式欧拉法 (Semi-implicit Euler)** 更新位置和速度。
*   **第3天：形状定义 (Collider)**
    *   实现 `Circle` 和 `AABB`（轴对齐包围盒）数据结构。
    *   计算质量、转动惯量（Moment of Inertia）。

#### 第2阶段：碰撞检测（第4-8天）
*   **第4天：圆 vs 圆 碰撞逻辑**
    *   计算两个圆是否重叠，生成 `CollisionManifold`（包含法线、深度）。
*   **第5-6天：SAT 算法（多边形碰撞）**
    *   手搓分离轴定理（Separating Axis Theorem）。
    *   **挑战**：找出两个矩形碰撞时的最小穿透向量。
*   **第7天：宽相 (Broadphase) 过滤**
    *   实现 AABB 碰撞预检查。如果两个物体的包围盒没撞，就不进行昂贵的 SAT 计算。
*   **第8天：日志系统**
    *   手搓一个简单的日志类，能把每一步的物理状态输出到控制台或文件。

#### 第3阶段：冲量响应与解算器（第9-12天）
*   **第9天：简单碰撞响应**
    *   基于冲量 (Impulse) 的速度更新。考虑恢复系数 $e$（弹性）。
*   **第10天：摩擦力与旋转**
    *   计算碰撞产生的转矩 (Torque)，让物体撞击后能转起来。
*   **第11天：穿透修正 (Position Correction)**
    *   解决“物体深陷”问题。即使速度降为0，也要把重叠的物体推开。
*   **第12天：多约束解算 (Sequential Impulses)**
    - **多点碰撞解算**：让木板能用“两只脚”稳稳站立。
	- **静摩擦与动摩擦**：让木板撞击后能立刻刹车，而不是在地上无休止地滑动。

#### 第4阶段：验证与工程化（第13-15天）
*   **第13天：单元测试 (Unit Tests)**
    *   **不调包**：自己写个简单的测试框架。
    *   测试：两个 1kg 的球 1m/s 对撞，撞完后速度是否符合物理规律。
*   **第14天：CSV 导出与分析**
    *   编写一个 `Exporter`，将物体轨迹导出来。你可以用 Excel 绘制出轨迹图，证明你的引擎是完美的。
*   **第15天：README 与 GitHub 整理**
    *   **重点**：在 README 里贴出你的测试数据、CSV 生成的图表，并详细说明你如何处理 $F=ma$ 的离散化。

### 这个项目在 GitHub 上的亮点是什么？
1.  **Test-Driven Development (TDD)**：由于没有画面，你被迫编写了大量的测试用例，这在企业开发中是极其重要的加分项。
2.  **数据驱动**：你可以实现一个简单的配置文件读取（比如读一个 `.txt`），根据配置初始化物理世界。
3.  **高性能库潜质**：你可以宣传这是一个“高性能、无依赖、可嵌入式”的物理运算库。


既然是“从0到1”手搓，项目结构的专业性（Professionalism）非常重要。这不仅能让你的 GitHub 仓库看起来不像学生作业，还能让你在编写过程中逻辑清晰。

### 2. 核心模块类名建议（按 15 天计划分布）

为了让你在 Visual Studio 里创建类时不迷茫，这里给出核心头文件的命名建议：

#### **A. Common 模块 (第1-2天)**
*   `Vector2.h`: 手搓 `Vec2` 类，重载运算符。
*   `Settings.h`: 存放全局常量，如 `GRAVITY = -9.81f`，`FPS = 60`。

#### **B. Dynamics 模块 (第3-4天，第9-12天)**
*   `Body.h`: `struct Body` 或 `class Body`。包含 `Position`, `Velocity`, `Mass`, `Inertia`。
*   `World.h`: 核心管理类。包含一个 `std::vector<Body*>`，负责执行 `Step(dt)`。

#### **C. Collision 模块 (第5-8天)**
*   `Shape.h`: 基类。
*   `Circle.h` / `Polygon.h`: 继承自 Shape。
*   `Manifold.h`: **非常重要**。用于存储碰撞发生时的信息（两个物体是谁、碰撞法线、穿透深度、碰撞点）。
*   `CollisionDetector.h`: 静态函数，专门处理 `CircleVsCircle`, `PolygonVsPolygon`。

#### **D. Utils 模块 (第13-14天)**
*   `CSVExporter.h`: 负责把每一帧物体的 `x, y` 坐标写入文件。
*   `Logger.h`: 简单的控制台彩色输出（提示：Collision Detected!）。

### 3. 给你的 Visual Studio 设置建议

1.  **头文件路径**：
    在 VS 项目属性中，进入 `C/C++ -> General -> Additional Include Directories`，添加 `$(ProjectDir)include`。
    *这样你在写代码时可以直接 `#include "physics/Common/Vector2.h"`，非常专业。*

2.  **C++ 标准**：
    在 `C/C++ -> Language -> C++ Language Standard` 中选择 **ISO C++11 Standard**。

3.  **防止代码混乱**：
    虽然你可以把所有代码写在 `main.cpp`，但**千万别这么做**。严格按照上面的 `src` 和 `include` 分离。
