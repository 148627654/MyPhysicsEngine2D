# Day 11: 位置修正与 Baumgarte 稳定化

## 1. 目标
解决物体在静止堆叠或低速碰撞时的“深陷”和“抖动”问题。通过直接修改物体的坐标（Position），将重叠的物体强制推开。

## 2. 核心概念：穿透补偿
由于数值计算误差和引力的持续作用，物体每一帧都会产生微小的重叠。
*   **Baumgarte 稳定化**：我们不一次性把重叠全部修复（这会导致物体爆炸式弹出），而是每一帧修复一部分（例如 20%~80%）。
*   **允许误差 (Slop)**：为了防止物体在静止时由于极小重叠而产生高频抖动，我们设定一个微小的“容忍区”（如 0.01 到 0.05 单位），只有重叠超过这个值才进行修正。

## 4. 技术实现细节 (Tips)

### 1. 为什么不修正旋转？
虽然旋转也会导致重叠，但在 2D 基础引擎中，通常**只修正平动位置**就足以解决 95% 的下陷问题。修正旋转的位置会导致极其复杂的非线性计算，容易引起系统崩溃。

### 2. 静态物体的优雅处理
由于你使用了 `invMass`，当 A 撞击地面时：
*   地面 `invMassB = 0`。
*   分母变成 `invMassA + 0`。
*   结果是地面不动，A 承担所有的位移修正。这完美符合物理直觉。

### 3. 防止“爆炸”
如果你的 `bias` 设为 `1.0`（100% 修复），物体可能会因为碰撞检测的一丁点误差而产生巨大的位移，看起来像爆炸。建议先从 `0.2` 开始。

### 💡 核心代码框架预览 (Solver.cpp)

```cpp
void PositionalCorrection(Manifold& m) {
    Body* A = m.bodyA;
    Body* B = m.bodyB;

    const float slop = 0.05f; // 穿透多少以内不修正
    const float percent = 0.4f; // 修正率 (20% - 80%)

    float invMassSum = A->getInvMass() + B->getInvMass();
    if (invMassSum < 0.0001f) return;

    // 只有当穿透深度大于 slop 时才进行修正
    float correctionMag = std::max(m.penetration - slop, 0.0f) / invMassSum * percent;
    Vector2 correctionVec = m.normal * correctionMag;

    A->SetPosition(A->GetPosition() - correctionVec * A->getInvMass());
    B->SetPosition(B->GetPosition() + correctionVec * B->getInvMass());
}
```
