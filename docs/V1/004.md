# Day 04: 基础碰撞检测与流形构造 (Circle vs. Circle)

## 1. 目标
实现圆与圆之间的碰撞检测算法，并构造 **Collision Manifold (碰撞流形)** 对象。该对象将存储“如何修复碰撞”所需的所有几何信息。

## 2. 核心概念：碰撞流形 (Collision Manifold)
在物理引擎中，仅仅知道“撞没撞”是不够的。为了让物体弹开，我们需要知道：
1.  **Normal (法线)**：碰撞发生的碰撞面方向（从物体 A 指向物体 B 的向量）。
2.  **Penetration Depth (穿透深度)**：两个形状重叠了多少。
3.  **Contact Points (接触点)**：碰撞发生的具体空间位置。
*这些信息的集合就称为“流形 (Manifold)”。*

---

## 3. 必须实现的功能清单

### A. 定义 CollisionManifold 结构 (Collision 模块)
- [ ] **Manifold 结构体**:
    - `Body* bodyA`: 参与碰撞的第一个物体。
    - `Body* bodyB`: 参与碰撞的第二个物体。
    - `Vector2 normal`: 碰撞法线（由 A 指向 B）。
    - `float penetration`: 穿透深度。
    - `std::vector<Vector2> contacts`: 接触点列表（圆圆碰撞通常只有一个点）。

### B. 实现碰撞检测逻辑 (CollisionDetector)
- [ ] **静态工具类/命名空间**:
    - 实现 `bool CircleVsCircle(Manifold* m, Body* a, Body* b)`。
- [ ] **检测算法**:
    1. 计算质心距离 $d = |PosB - PosA|$。
    2. 比较 $d$ 与两个圆的半径之和 $R_a + R_b$。
    3. 如果 $d < R_a + R_b$，则发生碰撞。

### C. 填充流形数据
如果发生碰撞，计算以下细节：
- [ ] **Normal**: $(PosB - PosA) / d$（注意处理 $d=0$ 的特殊情况，防止除零）。
- [ ] **Penetration**: $(R_a + R_b) - d$。
- [ ] **Contact Point**: $PosA + Normal \times R_a$（或者两圆心按半径比例插值）。

## 4. 技术实现细节 (Tips)

### 1. 性能小技巧：平方距离
在判断“是否碰撞”时，先使用距离的平方进行比较，避免昂贵的 `sqrtf()` 开销：
```cpp
Vector2 relativePos = b->position - a->position;
float distSq = relativePos.LengthSquared();
float radiusSum = rA + rB;

if (distSq > radiusSum * radiusSum) return false; // 没撞，直接返回
```

建议在 `CollisionDetector` 中使用这种结构，方便后续扩展：


### 3. 处理重合情况
如果两个圆心完全重合（$d=0$），法线会变成 `NaN`。
**策略**：如果 $d \approx 0$，手动给定一个人工法线，如 `Vector2(1, 0)`，并给出一个极小的穿透深度。

## 5. 今日验收测试 (Unit Test)

这是验证碰撞算法准确性的核心步骤：

1.  **相离测试**：
    - 圆 A(0, 0, r=1), 圆 B(3, 0, r=1)。
    - 验证：返回 `false`。
2.  **标准碰撞**：
    - 圆 A(0, 0, r=1), 圆 B(1.5, 0, r=1)。
    - 验证：返回 `true`，`penetration` 应为 `0.5`，`normal` 应为 `(1, 0)`。
3.  **垂直对撞**：
    - 圆 A(0, 0, r=2), 圆 B(0, 1, r=2)。
    - 验证：返回 `true`，`normal` 应为 `(0, 1)`，`penetration` 应为 `3.0`。
4.  **接触点验证**：
    - 确保 `contacts[0]` 的坐标落在两个圆的交界处。
