# Day 12: 顺序冲量 (Sequential Impulses) 与 摩擦力改进

## 1. 目标
让物体能够“稳如泰山”地堆叠，并实现真实的摩擦效果。今天我们将解决两个核心难题：多点碰撞引起的旋转抖动，以及让木板在斜面上能根据摩擦力停住。

## 2. 核心概念

### A. 迭代解算 (Sequential Impulses)
在之前的版本中，我们每帧只对每个碰撞对处理一次冲量。但在现实中，一个物体可能同时被多个物体挤压。
*   **核心思想**：通过多次迭代（Iterations），让系统逐渐趋向于稳定解。每一轮迭代都会微调物体的速度。
*   **为什么需要？**：如果你有 3 个木块堆叠，最底层的木块需要承受上面两个木块的重量。单次解算无法传递这种“压力”，只有多次迭代才能让冲量在物体间正确传播。

### B. 库仑摩擦力 (Coulomb Friction)
摩擦冲量（Tangent Impulse）的大小受限于法向冲量（Normal Impulse）：
*   **公式**：$|F_t| \le \mu \cdot F_n$
*   其中 $\mu$ 是摩擦系数。这意味着如果物体压得越紧（$F_n$ 越大），摩擦力就越大。

## 4. 技术实现细节 (Tips)

### 1. 为什么摩擦力需要法向冲量？
摩擦力是由于表面微观凸起相互阻碍产生的。如果一个球只是轻轻掠过地面，它的摩擦力极小；如果它被重重压在地上，摩擦力会巨大。因此，你**必须先计算法向冲量**，再根据它来限制切向冲量。

### 2. 区分静摩擦与动摩擦
简单起见，可以定义 `friction = sqrt(bodyA->friction * bodyB->friction)`。
*   如果想要更真实：`totalFriction = (vel_tangent < 0.01) ? staticFriction : dynamicFriction;`

### 3. 多点碰撞（Contact Clipping）
对于矩形 vs 矩形，由于可能存在“边对边”碰撞，会产生两个接触点。如果你只取中心点，木板会像不倒翁一样在中心点晃动。**两个接触点**能让木板产生力矩平衡，从而平稳落地。

### 💡 核心代码框架预览 (摩擦力部分)

```cpp
// 在 ResolveCollision 函数内部
for (int i = 0; i < m.contactCount; ++i) {
    // 1. 处理法向冲量 (与昨天类似，但要针对每个点)
    // ... calculate lambda_n ...
    
    // 2. 处理切向冲量 (摩擦力)
    Vector2 relativeVel = CalculateRelativeVelocity(m.bodyA, m.bodyB, m.contactPoints[i]);
    Vector2 tangent = relativeVel - (m.normal * Dot(relativeVel, m.normal));
    tangent.Normalize();

    float vt = Dot(relativeVel, tangent);
    float lambda_t = -vt / invMassSum;

    // 库仑定律限制：摩擦冲量不能超过 mu * 法向冲量
    float maxFriction = friction * m.normalImpulse[i];
    lambda_t = std::clamp(lambda_t, -maxFriction, maxFriction);

    // 应用切向冲量
    ApplyImpulse(m.bodyA, m.bodyB, tangent * lambda_t, m.contactPoints[i]);
}
```
