# Day 07: 宽相过滤与 AABB 系统

## 1. 目标
为所有形状引入 AABB 属性，并在 `World::Step` 中实现初步的碰撞过滤逻辑。将碰撞检测次数从 $O(N^2)$ 的几何运算降低为 $O(N^2)$ 的简单数值比较（并在未来支持更高级的空间分割）。

## 2. 核心概念：AABB (Axis-Aligned Bounding Box)
AABB 是一个不随物体旋转的矩形框，其边永远平行于坐标轴。
*   **计算速度极快**：判断两个 AABB 是否重叠只需要 4 次浮点数比较。
*   **空间占用**：它总是能包裹住物体。对于旋转的矩形，其 AABB 会随着旋转而变大或缩小。


    3.  如果**重叠**，再调用昂贵的 `Collision::Dispatch`。

## 4. 技术实现细节 (Tips)

### 1. 为什么 AABB 要放在 Body 里？
虽然 AABB 的计算逻辑属于 `Shape`，但 AABB 的具体数值属于 `Body`（因为它随位置变化）。在 `World` 遍历时，直接访问 `body->aabb` 比临时调用 `shape->ComputeAABB` 快得多。

### 2. 旋转矩形的 AABB 动态变化
注意：旋转 45 度的正方形，其 AABB 会比旋转 0 度时大（对角线长度）。
**千万不要**直接用 `width/height` 作为 AABB，必须用变换后的顶点坐标。

### 3. 性能监测 (Logging)
为了验证 Day 07 的成果，你可以在控制台打印：
`"Broadphase: Skipped X pairs, Narrowphase: Checked Y pairs"`。
你会发现对于散落在地图上的物体，检查次数 `Y` 会远小于原始的 $N(N-1)/2$。

```cpp
    // 1. 积分更新位置
    // ...

    // 2. 更新所有物体的 AABB

        body->UpdateAABB(); // 内部调用 shape->ComputeAABB

            if (Collision::Dispatch(&m, a, b)) {
                // 存储碰撞，等待后续处理
            }
        }
    }
}
```
