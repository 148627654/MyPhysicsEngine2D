## 2. 核心概念：为什么是“半隐式欧拉法”？

在计算机中模拟物理，我们需要将连续的时间切成一小块一小块的 `dt`（Delta Time）。
*   **显式欧拉 (Explicit Euler)**: 
    1. $p_{new} = p_{old} + v_{old} \cdot dt$
    2. $v_{new} = v_{old} + a \cdot dt$
    *缺点：非常不稳定，能量会迅速爆炸，球体会越弹越高。*

*   **半隐式欧拉 (Semi-implicit Euler)**:
    1. **$v_{new} = v_{old} + a \cdot dt$** (先算新速度)
    2. **$p_{new} = p_{old} + v_{new} \cdot dt$** (用新速度算位置)
    *优点：在简单物理模拟中具有很好的能量守恒性，是游戏引擎（如 Box2D）的标准做法。*

## 3. 必须实现的功能清单

### A. RigidBody 类 (Dynamics 模块)
创建一个 `Body` 类，目前先关注**线性运动**（暂时忽略旋转）：
- [ ] **物理属性**:
    - `Vector2 position`: 当前位置。
    - `Vector2 velocity`: 当前速度。
    - `Vector2 force`: 累积力（每一帧更新前清零）。
    - `float mass`: 质量。
    - `float invMass`: **质量的倒数** (性能优化关键！如果 `mass=0`，则 `invMass=0` 表示固定不动的物体)。
    - `float gravityScale`: 重力缩放系数（有些物体掉得快，有些不受重力影响）。
- [ ] **力学接口**:
    - `AddForce(Vector2 f)`: 累加力（$F = \sum f$）。
    - `ClearForce()`: 每帧结束清空力。

### B. World 类 (核心管理器)
- [ ] **容器**: 存储所有 `Body` 指针的集合 (`std::vector<Body*>`)。
- [ ] **配置**: 全局重力向量（例如 `Vector2(0, -9.81f)`）。
- [ ] **Step(float dt) 函数**:
    1. 遍历所有物体。
    2. 计算加速度 $a = F \cdot invMass$。
    3. 加上全局重力产生的加速度。
    4. 执行**半隐式欧拉积分**更新速度和位置。
    5. 清空物体的力。

### C. 全局设置 (Settings.h)
- [ ] 定义固定时间步长，例如 `DT = 1.0f / 60.0f`。

## 4. 技术实现细节 (Tips)

### 1. 质量倒数 (Inverse Mass) 的妙用
在物理引擎中，我们很少直接用 `mass`。
*   如果物体是**静态的**（比如地面），质量是无穷大。在代码里表示无穷大很难。
*   但无穷大的倒数是 `0`。
*   加速度公式 $a = F / m$ 变成 $a = F \cdot invMass$。如果 `invMass` 是 0，加速度就是 0，物体永远不动。这完美解决了地面的表示问题。

### 2. 积分代码伪代码
```cpp
// 在 World::Step(dt) 中
for (auto& body : bodies) {
    if (body->invMass == 0.0f) continue; // 静态物体跳过

    // 1. 计算加速度 (a = F/m + g)
    Vector2 acceleration = body->force * body->invMass;
    acceleration += gravity * body->gravityScale;

    // 2. 积分速度 (v = v + a*dt)
    body->velocity += acceleration * dt;

    // 3. 积分位置 (p = p + v*dt)
    body->position += body->velocity * dt;

    // 4. 清力
    body->ClearForce();
}
```
