# V2-Day 3: 树的自我平衡 (Tree Rotations)

## 1. 目标
解决动态 AABB 树的“退化”问题。当物体按顺序插入（如排成一排）时，树会变得像链表一样深。今天我们将实现 **AVL 旋转算法**，在插入节点的同时自动调整树结构，确保树的高度维持在 $O(\log N)$，从而保证碰撞查询的极致性能。
## 2. 核心概念
### A. 树的高度 (Height)
*   **叶子节点**的高度定义为 `0`。
*   **内部节点**的高度定义为 `1 + max(leftChild.height, rightChild.height)`。
*   当左右子树的高度差超过 `1` 时，这棵树就失去了平衡。
### B. AVL 旋转 (AVL Rotations)
在 AABB 树中，旋转并不是简单的数值交换，而是**子树的重新挂接**：
*   **核心思想**：如果右子树太深，我们就把右子树的一个孩子“提拔”上来，把原来的根节点“降级”为它的兄弟，并将其中一个孙子节点移交给降级后的节点。
*   **目的**：在不破坏 AABB 包含关系的前提下，减少树的总层数。

## 4. 技术实现细节 (Tips)

### 1. 旋转中的“孙子”转移
以“向左旋转”为例（右子树太深）：
假设当前节点是 A，右孩子是 C。C 有两个孩子 F 和 G。
若 F 的高度大于 G，我们将 A 的右孩子设为 F，C 的左孩子设为 A。
*注意*：必须更新 F 的 `parent` 为 A，A 的 `parent` 为 C。

### 2. AABB 的及时更新
旋转本质上改变了物理空间的嵌套关系。**先更新孙子辈的 AABB，再更新祖父辈的 AABB**，顺序绝对不能错。

### 3. 索引 vs 指针
在执行 `m_nodes[i].leftChild = j` 这种操作时，一定要非常小心，建议在纸上画出 A-G 节点的索引变化图。

### 💡 核心代码架构预览 (Balance 逻辑片段)

```cpp
int32_t DynamicTree::Balance(int32_t iA) {
    Node* A = m_nodes + iA;
    if (A->isLeaf() || A->height < 2) return iA;

    int32_t iB = A->leftChild;
    int32_t iC = A->rightChild;
    int32_t balance = m_nodes[iC].height - m_nodes[iB].height;

    // 右子树太深
    if (balance > 1) {
        int32_t iF = m_nodes[iC].leftChild;
        int32_t iG = m_nodes[iC].rightChild;
        
        // 交换 A 和 C
        m_nodes[iC].leftChild = iA;
        m_nodes[iC].parent = A->parent;
        A->parent = iC;

        // A 的老上司现在要认 C 当儿子
        if (m_nodes[iC].parent != -1) {
            if (m_nodes[m_nodes[iC].parent].leftChild == iA) m_nodes[m_nodes[iC].parent].leftChild = iC;
            else m_nodes[m_nodes[iC].parent].rightChild = iC;
        } else {
            m_root = iC;
        }

        // 孙子节点 F 和 G 谁高，谁就留在 C 下面，矮的移交给 A
        if (m_nodes[iF].height > m_nodes[iG].height) {
            m_nodes[iC].rightChild = iF; // 其实 C 的孩子已经是 F,G 了，这里是示意
            A->rightChild = iG;
            // ... 更新 parent 和 AABB ...
        }
        return iC;
    }
    // ... 左子树同理 ...
    return iA;
}
```
