# V2-Day 1: 动态 AABB 树的基础架构 (Node & Pool)

## 1. 目标
开始构建物理引擎的“大脑”——**动态 AABB 树 (Dynamic AABB Tree)**。今天我们将完成树的基础数据结构设计，实现一个基于“内存池”思想的节点管理系统，为后续 $O(\log N)$ 复杂度的碰撞查询打下基础。

## 2. 核心概念

### A. 空间分割与 BVH
在 V1 中，我们使用 $O(N^2)$ 的双重循环检查碰撞，当物体达到 100 个时，系统会显著变慢。
*   **BVH (Bounding Volume Hierarchy)**：通过将物体嵌套在更大的包围盒中形成树状结构。如果一个大的包围盒没撞，其内部成百上千个小物体都不用检查。
*   **动态树**：与静态的四叉树不同，动态树允许物体频繁移动、进入或离开，而不需要每一帧彻底重建。

### B. 节点池 (Node Pool)
*   **为什么要用数组而不是 `new` 指针？**：频繁创建和销毁节点会导致内存碎片和性能下降。
*   **索引管理**：我们使用 `int` 索引来代替 `Node*` 指针。这不仅方便内存扩容，而且对 CPU 缓存极其友好。
*   **空闲列表 (Free List)**：通过一个链表结构管理数组中未使用的槽位，实现 $O(1)$ 的节点分配和回收。

## 4. 技术实现细节 (Tips)

### 1. 索引标记
建议使用 `-1` (或定义 `NULL_NODE`) 来表示空索引。对于叶子节点，`child1` 和 `child2` 应该都设为 `-1`。

### 2. 空闲列表的巧妙实现
在数组中，如果一个节点是“空闲”的，我们可以利用它的 `next` 成员指向下一个空闲节点。
```cpp
// 初始化空闲列表
for (int32_t i = 0; i < m_nodeCapacity - 1; ++i) {
    m_nodes[i].next = i + 1;
}
m_nodes[m_nodeCapacity - 1].next = -1; // 最后一个
m_freeList = 0;
```

### 3. 为什么是二叉树？
虽然四叉树或八叉树也很流行，但在 2D 动态环境中，**二叉树**的维护成本（旋转平衡）最低，且在物体分布不均时表现最稳健。

### 💡 核心代码框架预览 (DynamicTree.h)

```cpp
struct Node {
    AABB aabb;
    void* userData; // 存储 Body*
    
    int32_t parent;
    int32_t child1;
    int32_t child2;
    
    // 用于空闲列表
    union {
        int32_t next;
        int32_t height; // 树平衡时使用
    };

    bool IsLeaf() const { return child1 == -1; }
};

class DynamicTree {
public:
    DynamicTree();
    ~DynamicTree();

    int32_t CreateProxy(const AABB& aabb, void* userData);
    void DestroyProxy(int32_t proxyId);

private:
    int32_t AllocateNode();
    void FreeNode(int32_t nodeId);

    Node* m_nodes;
    int32_t m_nodeCount;
    int32_t m_nodeCapacity;
    int32_t m_root;
    int32_t m_freeList;
};
```
